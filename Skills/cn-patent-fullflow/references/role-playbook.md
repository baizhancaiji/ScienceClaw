# Role Playbook

Use this file when a step benefits from a stronger specialist mindset, without reviving the original multi-agent packages.

## Technology Mining Lens

Apply when the user provides only a vague idea or scattered materials.

- Extract technical problem, technical solution, and technical effect.
- Expand one idea into multiple patentable angles.
- Force specificity where the description is vague.

## Prior-Art Search Lens

Apply when building keyword strategy and retrieval plan.

- Build keywords before searching.
- Expand synonyms and equivalent Chinese drafting terms.
- Record channels and failures clearly.
- Prefer breadth first, then narrow on risk.

## Drafting Lens

Apply when writing the main Chinese draft.

- Keep terminology consistent.
- Use non-code technical descriptions.
- Quantify technical effects where support exists.
- Ensure the specification can support later claim scope.

## Claims Lens

Apply when strengthening claim structure.

- Build a claim tree, not a random list.
- Use dependent claims as fallback positions.
- Do not claim beyond disclosed support.

## Audit Lens

Apply before calling a draft filing-ready.

- Check support, clarity, consistency, inventive-step positioning, and drawing alignment.
- Report critical defects before style issues.
- Give a filing-readiness judgment, not just edits.
